## B

```{glossary}

Binder
  ユーザーが定義した環境で、プロジェクトのバージョンを完全にアップロードして共有できるウェブベースのサービス。

Binderhub
  バインダーを生成するサービス。 最も広く使用されているのは、Binderチームによって管理されているmybinder.orgです。 より専門的な構成をサポートできる他のBinderHubを作成することが可能です。 そのような設定の1つには、プライベートリポジトリを密接な共同作業者間で共有できるようにするための認証が含まれる可能性があります。

バインダー化
  プロジェクトのバインダーを作成します。

ブランチ
  リポジトリの並列バージョン。 同じリポジトリ内に含まれていますが、個別に開発し、必要に応じて変更を「ライブ」リポジトリまたは他のブランチにマージすることができます。

バグ
  これはエラーです。 コンピュータプログラムやシステムの不具合や故障が原因で不正確な結果や予期せぬ結果が生じます 意図せぬ振る舞いをすることです


  ジョブのグループを構築します。 たとえば、ビルドには2つのジョブがあり、それぞれ異なるバージョンのプログラミング言語を持つプロジェクトをテストします。 すべてのジョブが完了すると、ビルドは終了します。
```

## C

```{glossary}

チェックアウト
  特定のファイル、ブランチ、またはコミットに切り替えるGitコマンド。 古いバージョンのファイルやコミットを有効化したり、アクティブなブランチを切り替えたりできます。

市民科学
  科学的研究に国民のメンバーを含める。


  既存の Git リポジトリをコピーし、通常はリモートの場所からローカル環境にコピーします。 リポジトリをクローンすると、その履歴全体とすべてのブランチをコピーします。

Code Coverage
  テストスイートによってソースコードがどの程度行使されているかを示す測定値。

行動基準
  行動の種類を確立するガイドライン は、コミュニティの中で奨励されました ガイドラインの問題または違反が解決されるプロセスと、それらの実施担当者の概要を説明します。

Code Review
  コード品質をテストする追加の方法。 コードレビューでは、別のプログラマーが新しいコードを調べて評価することができます。 目標は、強みと改善の潜在的な分野を指摘することです。

強制的な著者
  上級研究者が、贈り物やゲストの著者を含めるようにジュニア研究者を強制するとき。

プロジェクト履歴の
  スナップショットをコミットします。 コミットは、単一のファイルまたはファイルとディレクトリの範囲の変更後に行うことができます。

Commit Message
  ユーザがコミットに添付できるメッセージ。

コミュニケーションチャンネル
  メーリングリスト、コミュニティフォーラム、チャット、ソーシャルメディアを含む可能性のあるプロジェクトのために確立されたコミュニケーション方法。

コミュニティメンバー
  プロジェクトを利用する人。 会話で活発になったり、プロジェクトの方向性について意見を述べたりするかもしれません。

計算環境
  作業の挙動に影響を与えるコンピュータの特徴 例えば、オペレーティングシステム、インストールされているソフトウェア、インストールされているソフトウェアのバージョンなどです。

Conda
  一般的に使用されるパッケージ管理システム。

Consortium authorship
 集団またはコミュニティグループの著者モデル。 コンソーシアムのすべてのメンバーは著者と見なされ、時にはグループ名で公開されることがありますが、通常は公開された記事に記載する必要があります。 コンソーシアムのすべてのメンバーが著者の責任に同意しない場合。 著者であるメンバーは、そうでないメンバーとは別にリストされます。 

Container
  オペレーティングシステム、カスタマイズされた設定、ソフトウェア、ファイルを含む計算環境全体をカプセル化できる軽量ファイル。

継続配信
  プロジェクトのビルドとテストに必要なステップを自動化して実行します。

継続的なデプロイ
  コードが変更されるたびに自動的にデプロイされます。

継続的統合
  これは、個人が行ったプロジェクトへの変更を主に統合する練習です。 頻繁に共有されているバージョン(通常は1日に複数回)。 CIとも呼ばれる。

コントリビュートガイドライン
  オープンソースプロジェクトへの貢献方法を概説したガイドライン。

貢献者
  プロジェクトに貢献した全ての人。 これらは、研究の全体的な完成に貢献した研究プロジェクトのメンバーです。 これは、ソフトウェアでバグを修正したり、学術論文を執筆するなど、より大きな貢献をするなど、小さな貢献になる可能性があります。 

対応する著者
 研究グループの学術論文を管理する者。 査読者のコメント、証明書を受け取る責任があります。 編集者の情報が掲載されています

CRediTタクソノミ
 CRediTタクソノミは14のロールを含む高レベルタクソノミです。 科学的な成果の貢献者が果たす役割を表すために使用することができます。 役割は、各貢献者の学術成果への具体的な貢献を説明します。 これらの詳細は、著者の基準を満たすだけでなく、ジャーナルによってますます要求されるようになっています。

```

## G

```{glossary}
一般化可能
  複製可能と堅牢な発見を組み合わせることで、一般化可能な結果を形成することができます。 別のソフトウェア実装と異なるデータセットで分析を実行すると、一般的な結果は得られないことに注意してください。 研究の質問のすべての異なる側面に仕事がどのくらいうまく適用されるかを知るために、さらに多くのステップがあります. 一般化は、結果が特定のデータセットや特定のバージョンの解析パイプラインに依存しないことを理解するための重要なステップです。

Git
  GitHubが構築されているバージョン管理システム。 これは、Linuxの著者によって開発された、広く使用されているオープンソースの分散バージョン管理システムです。

Github
  オンラインコードホスティングとバージョン管理サービス。 これは、ユーザー間のコラボレーションを支援するための非常に多くの機能を備えており、多数のオープンソースプロジェクトをホストします。

GitLab
  GitLab is a web-based DevOps lifecycle tool that provides a Git-repository manager providing wiki, issue-tracking and continuous integration and deployment pipeline features, using an open-source license, developed by GitLab Inc.

Ghost author
 It is a person who writes an academic article without having carried out the research. それはプロの作家かもしれません。 彼らは、作家についてのICMJE基準の下で、作者としてしばしば資格を与えられなかった。

ギフト著者
 研究に大きな貢献をしなかった作者として記載されている方。 ゲスト作家としても知られています。

グループ authorship
 いくつかのジャーナルはグループ名の使用を許可していますが、多くは貢献者がリストされたり、書き込みグループに名前を付ける必要があります。 これは共有された著者と同じです。

Guarantor
 名付けられた著者であるための満たす基準と同様に、 一部の雑誌では、創業から出版までの間、作品全体の完全性に責任を持つ作者が必要とされています。
```

## M

```{glossary}

機械可読性
  機械可読性とは、文書、データ、またはその他のデジタル出力を指し、その内容はコンピュータによって容易に処理されます。 このような文書は、作成されたビジネスプロセスをサポートするために必要なコンテキストを提供するために十分な構造を有するために、機械読み取り可能なデータと区別されます。 機械読み取り可能なデータは、意味論的な意味が失われないことを保証しながら、人間の介入なしにコンピュータによって容易に処理できる形式のデータとして定義することができます。

メタデータ
  他のデータを記述するために使用されます。 例えば (35, 33, 27, 30) 33)はデータですが、ユニット(マイル/時)は、これらは特定の道路上の車の速度であるという事実はメタデータです。
```

```{glossary}
  特定のバージョンを含むソフトウェアパッケージをインストール、管理、アンインストールするためのツールです。

  コミュニティによって広く理解できるユニークでユニークなリソースを識別するための方法。

パターン
  パターンルールは、ターゲットに % 文字だけを含むルールです。 ファイル名の一部に合わせることができます。

ペルソナ
  ペルソナとは、架空のユーザーまたはメンバーの詳細なものです。 既存のメンバーや将来のメンバーの実際の観察と理解に基づいています。

ペルソナキャンバス
  ペルソナキャンバスは、すべての回答を一箇所にまとめるために使用できます。 あなたのメンタルモデル(私たちの考えから抽象概念)のこの有形の情報を同僚と共有し、あなたのコミュニティメンバーについてコミュニケーションするための共通言語を作成します。 ユーザーと貢献者です

Phony Target
  偽のターゲットは、ファイルシステム上のファイルに対応していないものです。 ターゲットは .PHONY ターゲットの前提条件として偽としてマークされます。

プロジェクト・デザイン
  プロジェクトの主要な機能、構造、成功基準、主要成果物がすべて計画される初期段階。

プルリクエスト
  リモートリポジトリへの変更提案。 書き込み権限のないコラボレーターは、リポジトリに加えた変更を含むプルリクエストを管理者に送信できます。 管理者は、メインリポジトリへの変更を承認およびマージまたは拒否できます。 オープンソースプロジェクトの場合、プロジェクトをフォークした人なら誰でもプルリクエストを送信できます。

プッシュ
  リモートリポジトリに変更を送信します。 リモートリポジトリはプッシュされた変更で更新され、ローカルリポジトリがミラーされるようになりました。
```

```{glossary}
レシピ
  Makeによって実行される1つ以上のシェルコマンド。 通常、これらのコマンドはルールのターゲットを更新します。

回帰テスト
  コードが変更された前後のテストの結果を比較する。 出力が変更された場合、プログラムのどこかで問題が発生し、エラーが発生します。

複製可能
  異なるデータセットで同じ解析を行った場合、結果が複製可能である。

repo2docker
  コード リポジトリから Docker イメージをビルドするためのツール。

リポジトリ
  *データまたはコード再現*と同じです。 リソース(データ、ソフトウェア、出版物など)が保存されアクセスされるインターネット上の長寿命の場所。 このキーワードはしばしば「repo」に短縮されます。

再現可能
  同じデータセットで同じ解析ステップが一貫して同じ解答を生成する場合、結果が再現可能である。

レンダリングされた出力
  GithubまたはWebページのオンラインページでテキストがどのように見えるか

Research Compendia
  これは、データを含む研究プロジェクトのすべてのデジタル部分のコレクションです コード、テキスト (プロトコル、レポート、アンケート、メタデータ) コレクションは、すべての結果をそのまま再現するように作られています。

研究データ管理
  *アクロニム: RDM* 研究プロジェクトで作成されたデータの組織、保存、保存を参照します。 初期計画、日々のプロセス、長期的なアーカイブと共有について説明します。 RDM に短縮しました。

研究倫理
  研究倫理とは、研究者がどのように仕事を遂行すべきかを左右する道徳的原則である。 これらの原則は、大学統治機関、コミュニティ、政府などのグループで合意された研究規制を形成するために使用されます。 すべての研究者は自分の仕事に適用される規制に従う必要があります。

レビュー
  変更を提案するか、すでに作成されているプルリクエストに何かをコミットするよう求めています。

リスク評価
  プロジェクトに適した持続可能なソフトウェアコンセプトを選択するために使用されます。

リスク行列
  リスク行列は、あなたが興味を持っていることで何が起こっているかを定量化する方法です。 一方の軸は何らかの方法で露出を測定し、もう一方は事故の影響を測定します。 起源から離れるほど、リスクを受け入れるためには、より多くの安全対策が必要です。

ロードマッピング
  プロジェクトのロードマップを作成します。 それはあなたがする必要がある仕事の概要である。 それはあなたの目標、ビジョン、タスクのスケジュールをカバーしています。


ロバスト
  同じデータセットが異なる解析ワークフローにさらされて同じ研究の質問(たとえばRで書かれたパイプラインとPythonで書かれた別のパイプライン)に答えられ、質的に類似または同一の答えが生成された場合、結果は堅牢です。 堅牢な結果は、解析を行うために選択されたプログラミング言語の特異性に依存しないことを示している。

ルール
  ビルドする必要があるものを定義する Makefile の要素は、通常、ターゲット、レシピ、およびオプションとして前提条件で構成されます。

Runtime Test
  プログラム内に埋め込まれたテスト。
```

```{glossary}
  プロジェクトホールドを実行するために必要な基本的な要件を確実にする非常に簡単な初期チェック。 これらが失敗した場合、修正されるまで追加のレベルのテストに進むことはありません。

構文
```
