## :gift: Ihr Beitrag zum Projekt

**Bitte lesen und folgen Sie unserem Projekt [Mitwirkende Richtlinien](https://github.com/alan-turing-institute/the-turing-way/blob/main/CONTRIBUTING.md)!** :sparkles:

Wenn Sie sie verwirrend, schwer zu verfolgen oder der Meinung sind, dass Informationen hinzugefügt werden müssen, lassen Sie es uns bitte wissen! Dies wird uns helfen, die Erfahrungen aller Beteiligten zu verbessern.

### Verwaltung Ihrer Beiträge zum GitHub Repository

Die tl;dr Version für einfache Referenz während des Bindestrichs:
1. Kommentieren Sie ein [bestehendes Ticket](https://github.com/alan-turing-institute/the-turing-way/issues) oder öffnen Sie ein neues Problem, das Ihren Zusatz referenziert.
    * Die [Ticket Vorlage](https://github.com/alan-turing-institute/the-turing-way/blob/main/ISSUE_TEMPLATE.md) wird automatisch im Kommentarbereich der neuen Ticket Seite angezeigt so dass Sie nur die Platzhaltersätze bearbeiten müssen.
    * Fügen Sie das `Buch-Dash-<location>` Label in der Spalte rechts ein.
2. [Fork](https://help.github.com/articles/fork-a-repo) das Turing Way Repository an dein GitHub Profil.
3. Änderungen vornehmen!
4. Senden Sie einen [Pull-Request](https://help.github.com/articles/creating-a-pull-request).
    * Öffnen Sie so früh wie möglich in Ihrem Beitragsprozess eine Pull-Request-Anforderung und fügen Sie das Label [WIP] zu dem Titel hinzu, um "In Bearbeitung arbeiten" zu bestimmen.
    * Die [Pull-Request-Vorlage](https://github.com/alan-turing-institute/the-turing-way/blob/main/.github/PULL_REQUEST_TEMPLATE.md) wird automatisch im Kommentar-Abschnitt der neuen Pull-Request-Seite angezeigt so dass Sie nur die Platzhaltersätze bearbeiten müssen.
    * Fügen Sie das `Buch-Dash-<location>` Label in der Spalte rechts ein.
    * Wenn Sie weiterhin Änderungen vornehmen, werden diese automatisch in Ihre bestehende Pull-Anfrage aufgenommen.
    * Ändere [WIP] auf [Bereit zur Überprüfung] im Pull-Request-Titel, wenn du glücklich bist, dass dein Beitrag überprüft und zusammengeführt wird. :tada:

**Welche Art von Beiträgen können Sie leisten?**

* Prüfer für Ihre Buch-Dash-Anwendung haben einige spezifische Themen für jeden von Ihnen vorgeschlagen
* Du kannst auch mit anderen zusammenarbeiten oder langsam anfangen, indem du die Liste der [offenen Tickets ansieht](https://github.com/alan-turing-institute/the-turing-way/issues)
* Korrektur [existierende Kapitel](https://the-turing-way.netlify.com) (Suche nach Typos, Grammatik und Links)
* Weitere Informationen zu einem existierenden Kapitel hinzufügen
* Themen für neue Kapitel vorschlagen (besonders wenn du sie schreiben kannst!)
    * Ein Kapitel kann mindestens 3 Absätze lang sein - worauf können Sie 3 Absätze schreiben?
    * Welche Fähigkeiten oder Werkzeuge wünschen Sie, dass Sie zu Beginn Ihrer Forschungslaufbahn unterrichtet wurden?
* Bewertung `[Bereit zur Überprüfung]` [Pull-Requests](https://github.com/alan-turing-institute/the-turing-way/pulls)
* Beitrag zur Diskussion in `[WIP]` Pull-Anfragen und Probleme
* Incorporative Fallstudien in das Buch
* Checklisten für spezifische Fähigkeiten oder Empfehlungen
* Restrukturierung der Kapitel (wie das Kapitel Reproduzierbarkeit)
* Kreative, aus der Kiste Ideen!
* Lassen Sie uns wissen, wenn Sie mit einem Beitrag in irgendeiner Weise kämpfen, damit wir unsere Mitwirkenden Richtlinien verbessern können!
