# 清单和阅读建议

## Checklist

### 自动格式化代码

- [ ] 将您的开发代码写入您最喜欢的 IDE/文本编辑器。
- [ ] 通过调整首选项/设置，在您的编辑器中启用自动格式化。
- [ ] 输入 `Ctrl + s` (windows, linux) 或 `D + s` (mac) 来保存工作以格式化代码。

### 用于静态代码分析

- [ ] 构建项目以启用 `linters` 来发现代码中的错误/警告(如果有的话)。
- [ ] 做出相关修改并重复上述步骤。
- [ ] {ref}`提交并推送<rr-vcs-git-commit>` 对远程 **Github/GitLab/BitBucket** 用于运行部署前测试的存储库。
