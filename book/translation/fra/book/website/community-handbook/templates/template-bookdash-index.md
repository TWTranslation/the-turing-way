## :gift: Votre contribution au projet

**Veuillez lire et suivre notre projet [Directives de contribution](https://github.com/alan-turing-institute/the-turing-way/blob/main/CONTRIBUTING.md)!** :sparkles:

Si vous les trouvez confus, difficiles à suivre ou si vous pensez que des informations doivent être ajoutées, veuillez nous le faire savoir ! Cela nous aidera à améliorer l'expérience de contribution pour tous ceux qui vont de l'avant.

### Gérer vos contributions au dépôt GitHub

La version tl;dr pour une référence facile pendant le dashing:
1. Commenter sur un [problème existant](https://github.com/alan-turing-institute/the-turing-way/issues) ou ouvrir un nouveau ticket référant votre ajout.
    * Le modèle de ticket [](https://github.com/alan-turing-institute/the-turing-way/blob/main/ISSUE_TEMPLATE.md) sera automatiquement affiché dans la section commentaire de la nouvelle page de ticket, donc tout ce que vous avez à faire est de modifier les phrases réservées.
    * Ajouter le label `book-dash-<location>` dans la colonne de droite.
2. [Fork](https://help.github.com/articles/fork-a-repo) le dépôt de Turing Way vers votre profil GitHub.
3. Faites vos changements !
4. Soumettez une [pull request](https://help.github.com/articles/creating-a-pull-request).
    * Ouvrez une pull request dès que possible dans votre processus de contribution et ajoutez l'étiquette [WIP] au titre pour désigner "work in progress".
    * Le modèle [pull request](https://github.com/alan-turing-institute/the-turing-way/blob/main/.github/PULL_REQUEST_TEMPLATE.md) sera automatiquement affiché dans la section commentaire de la nouvelle page de pull request donc tout ce que vous avez à faire est de modifier les phrases réservées.
    * Ajouter le label `book-dash-<location>` dans la colonne de droite.
    * Au fur et à mesure que vous continuez à apporter des modifications, elles seront automatiquement incluses dans votre pull request existante.
    * Changez [WIP] pour [Ready for review] dans le titre de la pull request lorsque vous êtes heureux que votre contribution soit révisée et fusionnée. :tada:

**Quels types de contributions pouvez-vous faire ?**

* Les réviseurs de votre application de tableau de bord du livre ont suggéré quelques sujets spécifiques pour chacun de vous
* Vous pouvez également collaborer avec d'autres personnes ou commencer lentement en vérifiant la liste des [tickets ouverts](https://github.com/alan-turing-institute/the-turing-way/issues)
* Relire [chapitres existants](https://the-turing-way.netlify.com) (vérifiez les fautes de frappe, la grammaire et les liens)
* Ajouter des informations supplémentaires à un chapitre existant
* Suggérer des sujets pour de nouveaux chapitres (surtout si vous pouvez les écrire !)
    * Un chapitre peut comporter au moins 3 paragraphes de longueur - sur quoi pouvez-vous écrire 3 paragraphes?
    * Quelles compétences ou quels outils souhaitez-vous que l'on vous enseigne au début de votre carrière de recherche?
* Examiner `[Prêt à réviser]` [demandes d'ajout](https://github.com/alan-turing-institute/the-turing-way/pulls)
* Contribuer à la discussion dans `[WIP]` demandes d'ajout et tickets
* Intégrer les études de cas dans le livre
* Listes de contrôle pour une compétence spécifique ou une recommandation
* Restructuration des sections du chapitre (comme le chapitre Reproductibilité)
* Créatif, des idées inédites !
* Faites-nous savoir si vous avez du mal à contribuer d'une manière ou d'une autre afin que nous puissions améliorer nos lignes directrices de contribution!
