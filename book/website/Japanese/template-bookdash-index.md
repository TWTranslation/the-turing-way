### GitHubリポジトリへの貢献を管理する

ダッシュ中に簡単に参照できるようにするtl;drバージョン:
1. [既存の課題](https://github.com/alan-turing-institute/the-turing-way/issues) にコメントするか、追加を参照する新しい課題を開きます。
    * [課題テンプレート](https://github.com/alan-turing-institute/the-turing-way/blob/main/ISSUE_TEMPLATE.md) は、新しい課題ページのコメント欄に自動的にレンダリングされます。 プレースホルダーの文章を編集すればいいのです
    * 右側の列に `book-dash-<location>` ラベルを追加します。
2. [Fork](https://help.github.com/articles/fork-a-repo) the Turing Way repository to your GitHub profile.
3. 変更を加えましょう！
4. [プルリクエスト](https://help.github.com/articles/creating-a-pull-request) を送信します。
    * 可能な限り早期にあなたの貢献プロセスのプルリクエストを開き、「作業中」を指定するためにタイトルにラベル [WIP] を追加します。
    * [プルリクエストテンプレート](https://github.com/alan-turing-institute/the-turing-way/blob/main/.github/PULL_REQUEST_TEMPLATE.md) は、新しいプルリクエストページのコメントセクションに自動的にレンダリングされます。 プレースホルダーの文章を編集すればいいのです
    * 右側の列に `book-dash-<location>` ラベルを追加します。
    * 変更を続けると、既存のプルリクエストに自動的に含まれます。
    * 投稿のレビューとマージに満足している場合は、プルリクエストのタイトルで [WIP] をformat@@1に変更します。 :tada:

**どんな種類の貢献ができますか？**

* ブックダッシュアプリケーションの査読者が、それぞれの特定のトピックを提案しています
* [未解決の問題](https://github.com/alan-turing-institute/the-turing-way/issues)のリストをチェックすることで、他の人とコラボレーションしたり、ゆっくり始めたりすることもできます。
* 既存のチャプターを [校正する](https://the-turing-way.netlify.com) (タイポ、文法、リンクを確認)
* 既存の章に詳細情報を追加
* 新しいチャプターのトピックを提案してください（特に書き込める場合は！）
    * 章の長さは最低3段落にすることができます - あなたは3段落を書くことができますか?
    * 研究キャリアのスタート時に教えられていたスキルやツールはどのようなものですか?
* レビュー `` [プルリクエスト](https://github.com/alan-turing-institute/the-turing-way/pulls)
* `[WIP]` で議論に貢献する
* 本にケーススタディを組み込んでください
* 特定のスキルやおすすめのチェックリスト
* 章セクションの再構築 (再現性章のような)
* ボックスのアイデアから創造的!
* 私たちの貢献ガイドラインを改善するために、何らかの形で貢献に苦労している場合は、お知らせください!
